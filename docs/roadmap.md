
# Roadmap

## Phase 1: Foundation

Goal: Create a professional, well-documented project skeleton.

Tasks:

- Define project mission
- Define MVP scope
- Create repository structure
- Add documentation
- Add sample job data
- Add initial skill taxonomy
- Create FastAPI skeleton
- Create Streamlit skeleton

Status: In progress

## Phase 2: Rule-Based MVP

Goal: Build the first working end-to-end workflow using simple explainable logic.

Planned features:

- Load sample jobs
- Detect skills using taxonomy keyword matching
- Accept user skill input
- Compare job skills with user skills
- Show matched and missing skills
- Generate simple learning recommendations
- Show ATS checklist

## Phase 3: Improved Data and Evaluation

Goal: Make the system more realistic and measurable.

Planned features:

- Add more manually curated sample jobs
- Improve skill taxonomy
- Add role categories
- Add tests for skill matching
- Create evaluation examples
- Document known limitations

## Phase 4: CV Support

Goal: Add candidate CV/profile analysis carefully and ethically.

Planned features:

- CV text input
- Basic CV section detection
- ATS-readability checklist
- Skill extraction from CV text
- Explainable feedback

Privacy note:

CV support should avoid storing personal data by default.

## Phase 5: Recommendation Engine

Goal: Improve recommendations while keeping them explainable.

Planned features:

- Skill priority scoring
- Role-specific learning paths
- Project recommendations
- Resource suggestions
- Skill grouping by importance

## Phase 6: Advanced AI Features

Goal: Add advanced AI only after the basic workflow works.

Possible features:

- NLP-based skill extraction
- LLM-assisted explanations
- Semantic skill matching
- German and English job description support

## Phase 7: Deployment and Portfolio Polish

Goal: Make the project easy for recruiters to explore.

Planned features:

- Deployed demo
- Screenshots
- Demo video
- Example use cases
- Clear limitations
- Technical blog post
